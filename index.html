Analyzing Common Chess Openings Across Various Elo Brackets¶

By Seyed Mohammad Ghaemi

Introduction¶

In this tutorial, we will be going through the data science pipeline

  • Data Collection
  • Data Management/Representation
  • Exploratory Data Analysis
  • Hypothesis Testing
  • Communication of Insights

in order to gain an understanding of if and how certain factors, specifically ones such as Elo ranking, determine the opening moves of competitive chess players. The ultimate goal is to find out whether there is a significant difference between the common openings players of different elo's use.

Here is a list of assumptions made regarding what the reader already knows:

  • An understanding of basic Python features
  • An understanding of basic Pandas library features
  • An understanding of basic Matplotlib library features
  • An Understanding the fundamentals of chess and opening theory
  • The ability to learn from documentation

Data Collection and Management¶

In this section, we will first gather our chess data Lichess data set, then store features which we care about, and finally visualize a few examples along the way.

Libraries¶

Here is a list of libraries we will be using and why we need them:

  • We need os.path for basic path operations
  • We need the urllib library to download from our chess database
  • The chess database will give zipped files in bz2 format, so we will need the bz2 library to unzip them
  • Each game in the database is stored as a PNG, the python-chess library will allow us to obtain the information stored in them
  • Python's Pandas library is needed in order to store and manage our chess data.
  • Scipy's stats library allows us to apply different hypothesis tests to our data

Note

Chess database background information¶

Lichess is a popular, opensource chess server that allows users to play online chess games as well as a variety of other chess-related activities. You can explore their site by clicking here.

They have an open database of chess games that were played on their site, stored in monthly format, these can be found here.

The files are zipped using bz2 and games are formatted as PGN files. PGNs are one of the most popular standards when it comes to representing/storing chess games. You can learn more about them from here, however, we will be covering what information they store later in the tutorial.

Getting the data¶

For the purposes of this tutorial, we will download the September 2014 data from the Lichess database site. I chose that particular dataset as it is only 159 MB which is quite low compared to the most recent ones which are 27 GB. We will still have just over a million games to analyze which is plenty. Once we have downloaded the data, we will unzip it using bz2.

Using the python-chess library¶

Now that we have our PGN data, we will use the python-chess library to obtain the information we are currently looking for, mainly the opening for each game, and the elo it happened in.

Once we obtain the PNG using the library, we can get the information we need by getting the header for each game. Headers consist of information about the chess game such as "BlackElo", "WhiteElo", "ECO", "Opening", "Date", and more. The pgn.read_game() command can be used to get the information behind an entire game. The problem with this is that we do not necessarily want how the entire game played out given to us, especially since we will be reading more than a million matches! This is where the pgn.read_headers() method becomes useful as it allows us to retrieve the header information from the current game.

More information on the various methods in the library can be found in the documentation here.

Note

Note that the PGNs are stored in a stack format and each time we call one of the read commands, the PGN or game at the top of the stack is taken out and given to us.

Formatting the Pandas data¶

Note that there are multiple flaws with our current table that will make it harder to process later on. These are:

  • Some values in each category are missing and as such are marked with "?", so we must figure out how to handle those cases.
  • The elo data is of type string when it should be an integer
  • We will want to group the elo data into intervals
  • When grouping into intervals, we may want to create binary columns where a 0 means the row is not in an interval, and a 1 means the data is in an interval
  • We may want to combine the White and Black elo into one group through techniques like averaging to make analysis easier

First let's handle the missing data since we will not be able to turn "?" into integers in the case of elo without doing this step. Let's look at different scenarios where the data is missing for each category as a way to help us figure out which technique will serve us best.

Continuing from that, maybe we also want to see if there are certain openings that are unique to specific elo intervals. For this, we will use the Term Frequency-Inverse Document Frequency (TF-IDF) algorithm. This is a technique that, for every term in a document, determines whether that term is unique to this document (occurs multiple times in the document and not many times in other documents). In our case, the documents are our 5 intervals and the terms are the opening codes. You can learn more about this statistical tool here.

In [470]:
# All unique ECO openings
all_eco = set(info_tbl_int["ECO"].values)
# One counter dictionary for each interval
dicts = [{key: 0 for key in all_eco} for i in range(5)]

i = 0
# Go through each interval and add to the counter of the appropriate dict's opening
for k, tbl in info_tbl_int.groupby("Int_AvgElo"):
    for val in tbl["ECO"].values:
        dicts[i][val] += 1
    i += 1

# Calculates the term frequency
def tf(freqs, doc_i, opening):
    # Term frequency of opening appearing in the doc_i interval
    tf = freqs[doc_i][opening] / sum(freqs[doc_i].values())
    # We take the log to reduce outliers
    return np.log(1 + tf)

# Calculates the inverse document frequency
def idf(freqs, opening):
    num_doc = len(freqs)
    # Number of intervals/docs opening appears in
    appears = 0
    for doc in freqs:
        appears += 1 if doc[opening] != 0 else 0
    return np.log(num_doc / appears)

# Calculates the tf-idf
def tf_idf(freqs, doc_i, opening):
    return tf(freqs, doc_i, opening) * idf(freqs, opening)

# Testing our tfidf, this should give us 0 since A00 appears in all intervals
tfidf = tf_idf(dicts, 0, "A00")
print("TF-IDF of opening A00 in doc 0 is", tfidf)

Hypothesis Testing¶

Say we want to know if there is a significant statistical difference between how the openings are distributed in the beginner elo levels versus the higher elo levels. For this job, we will be using the Mann-Whitney U Test which is a hypothesis test that is used to detect differences between two datasets. Learn more about this test here.

Our null hypothesis is that there is no significant difference between the first and last interval's opening distribution. We will be using the default alpha level of 0.05 since we are only analyzing chess elo levels and people's lives are not dependent on this so a higher confidence is not needed.

First we will get a histogram for each of the intervals, then we will utilize scipy.stats' mannwhitneyu method to calculate our p-value.
